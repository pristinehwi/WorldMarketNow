## 1. 정체성 및 목적

> 한국인 제도권 투자자가 접속한 그 시각의 시의성에 맞는 글로벌 시장 인과 구조를 파악하는 도구.
> 동시에 AI 활용 개발력 전시.

- **타깃**: 금융 전문성을 가진 한국인 제도권 투자자
- **핵심 가치**: 이슈 나열이 아닌 신호들 사이의 인과 구조와 연결
- **차별점**: 텔레그램 등 기존 도구가 못 하는 시각 적응형 인과 시각화
- **중요**: 업무망 이용자 비중 높음 → 모바일 접속이 필수적

## 2. 시각 적응형 3레이어 구조

어느 시각에 접속해도 동일한 논리로 동작:

```
[레이어 1 — 배경]
현재 열린 시장의 최근 추세
→ "판이 어떻게 깔려 있었나"

[레이어 2 — 인계 신호]
직전 마감 시장 → 현재 열린 시장으로 넘어온 맥락
→ "어떤 신호를 받아서 시작했나"

[레이어 3 — 현재]
지금 진행 중인 장중 흐름
→ "그 신호를 받아서 어떻게 반응하고 있나"
```

시장 스케줄 (KST 기준):
```
한국: 09:00~15:30
일본: 09:00~15:30
유럽: 17:00~01:30
미국: 23:30~06:00
```

갱신 주기: **30분마다 1회 서버 생성 후 캐싱. 장중 포함 허용.**

## 3. Claude API 4-콜 아키텍처

```
0번 콜 — 현재 시각 + 열린/마감 시장 인식
         3레이어 데이터 기반
         주목할 스레드 우선순위 및 개수 판단
         각 스레드 주제 윤곽 결정

1번 콜 — 0번 판단을 인풋으로
         각 스레드별 인과구조 JSON 초안 생성
         각 스레드별 적정 프리퀀시 판단

2번 콜 — 초안 비판적 검토
         인과 논리 / 프리퀀시 적절성 / 빠진 연결고리

3번 콜 — 최종 확정
         스레드 간 중요도 순위 반영
         각 스레드별 브리핑 텍스트 생성
```

## 4. 인과 그래프 구조

- **DAG** (방향성 비순환 그래프)
- 합류 / 분기 / 체인 3가지 패턴 혼합
- **멀티 스레드**: 오늘의 인과 흐름을 독립적 여러 스레드로 분리
- 각 노드에 **출처 명시** → 할루시네이션 투명화
- 한국 파급경로가 자연스러운 스레드에 한해 한국 시장 변수를 종착 노드로 (강제 아님)
- 시각화: **D3.js**

### DAG JSON 구조 예시
```json
{
  "threads": [
    {
      "id": "thread_001",
      "title": "달러 강세 재점화",
      "priority": 1,
      "frequency": "WEEKLY",
      "nodes": [
        {"id": "n1", "label": "연준 매파 발언", "source": "Reuters RSS 03:21 KST"},
        {"id": "n2", "label": "미 고용 서프라이즈", "source": "FRED API"},
        {"id": "n3", "label": "달러 강세 (DXY +0.8%)", "source": "Finnhub"},
        {"id": "n4", "label": "원/달러 급등", "source": "한국은행 API"},
        {"id": "n5", "label": "외국인 매도 압력", "source": "KRX"}
      ],
      "edges": [
        {"from": "n1", "to": "n3"},
        {"from": "n2", "to": "n3"},
        {"from": "n3", "to": "n4"},
        {"from": "n4", "to": "n5"}
      ],
      "briefing": "연준 발언과 고용 서프라이즈가 겹치며 달러 강세가 재점화됐습니다..."
    }
  ],
  "headline": "달러 강세가 모든 것을 압도하는 아침",
  "generated_at": "2026-04-01T08:30:00+09:00"
}
```

## 5. 멀티 프리퀀시 타임라인

```
OVERNIGHT / WEEKLY / MONTHLY / CUSTOM
```

- Claude가 각 스레드별 적정 프리퀀시 **자동 배정** (기본값)
- 사용자가 원할 경우 다른 프리퀀시로 수동 전환 가능 (추가 탐색 옵션)
- UI에서 Claude 자동 배정값임을 명시
- 타임라인 = 단순 UI가 아닌 **인과 오판 방지 데이터 레이어**
- 장중 타임스탬프로 "종가 스냅샷 함정" 방지

---

## 6. 타임라인 ↔ 인과 그래프 연동 (핵심 WOW)

**완전 연동 (A안)**:
```
타임라인 이벤트 클릭
→ 인과 그래프가 그 시점까지만 점진적 활성화
→ 이후 노드는 흐리게 표시
→ 스토리가 시간순으로 전개되는 경험
```
WOW factor 최우선. 구현 난이도 높지만 타협 없이 구현.

## 7. 화면 레이아웃

### PC
```
[최상단 — HEADLINE ZONE]
Claude 생성 "오늘의 한 문장" 타이핑 애니메이션
+ 스레드 썸네일 (크기/색상/미니DAG로 중요도 차별화)

[중단 — MAIN ZONE]
좌: 인과 그래프 (DAG, D3.js)
우: 멀티 프리퀀시 타임라인

[우단 — SIDE PANEL]
스레드 클릭 시 펼쳐지는 사이드 패널
관련 뉴스 + 지표 + 심화 데이터 표시

[하단 — MAP ZONE]
지정학/지역 스레드 활성 시에만 지도 표시
비활성 시 축소 또는 숨김
```

### 모바일 (별도 UX — A안)
```
[최상단]
오늘의 한 문장 (간결하게)

[메인]
스레드 카드 리스트 (중요도 순)
각 카드: 핵심 인과 플로우 텍스트 + 간단한 노드 표시

[탭]
DAG 전체 보기 (보조적 접근)
타임라인 보기
```

## 8. 데이터 레이어 설계

**원칙: 팩트 수집(파이프라인)과 해석(Claude) 분리**

### 무료 데이터 소스 (현재 확정)
```
가격/지수/환율/인트라데이 : yfinance, Finnhub 무료 (60회/분)
경제지표 + 캘린더 + 컨센서스: FRED API + Finnhub 무료
뉴스 헤드라인: RSS (Reuters 등)
지정학 이벤트: GDELT
한국 금리/환율: 한국은행 Open API
```

### Claude 웹서치 보완 (2순위)
```
롱테일 자산 맥락 보완용
인과 구조 주재료 아닌 참고자료로만
출처 명시 필수
```

### 향후 업그레이드 옵션
```
EODHD EOD+Intraday ($29.99/월)
→ GAS 데이터 수집 함수만 교체
→ 전체 설계 무변경
```

### 히스토리 맥락 원칙
```
수집 가능한 범위 내 최대한 넓은 시간 구간 확보.
유의미한 구간은 Claude가 스레드별로 판단.
고정 구간 없음.
```

## 9. 한국인 입맛 레이어

전 단계에 스며드는 구조. 고정 목록 아닌 원칙 기반 확장형:

```
[데이터 수집]
"한국 제도권 투자자 관점에서 글로벌 변수가
한국 시장과 연결될 수 있는가"를 기준으로
유연하게 확장되는 집합
(KOSPI/KOSDAQ/원달러/외국인수급/SOX 등은 최소 예시)

[Claude 해석]
프롬프트에 한국 파급경로 명시 지시

[그래프]
자연스러운 스레드에만 한국 종착 노드 (강제 아님)

[텍스트]
한국어 + 한국 시장 관행 표현
```

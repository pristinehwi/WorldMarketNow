WorldMarketNow 개발 로드맵

Phase 1 백엔드 파이프라인 (기반)

1-1. GAS 프로젝트 생성 + GitHub 연동 설정
1-2. 데이터 수집 모듈 작성
     (yfinance → Python → GAS 또는 Finnhub 직접 호출)
1-3. 시장 스케줄 인식 모듈
     (현재 시각 기준 열린/마감 시장 판단)
1-4. 4-콜 Claude API 파이프라인
1-5. 결과 JSON → GitHub push 자동화
1-6. 30분 트리거 설정 + 테스트

Phase 2 PC 프론트엔드 핵심
2-1. React 프로젝트 생성 + GitHub Pages 배포 설정
2-2. 기본 레이아웃 (HEADLINE / MAIN / MAP ZONE)
2-3. HEADLINE ZONE
     오늘의 한 문장 타이핑 애니메이션
     스레드 썸네일 카드 (크기/색상 중요도 차별화)
2-4. DAG 인과 그래프 (D3.js)
     노드/엣지 렌더링
     합류/분기/체인 패턴
2-5. 타임라인 구현
2-6. 타임라인 ↔ DAG 완전 연동 애니메이션 ← 핵심 WOW
2-7. 사이드 패널 (스레드 클릭 시)
2-8. 지도 (조건부 표시, Leaflet.js)

Phase 3 모바일 프론트엔드
3-1. 모바일 레이아웃 설계
3-2. 스레드 카드 리스트
3-3. 단순화된 인과 플로우
3-4. DAG 탭 접근
3-5. 반응형 분기 처리

Phase 4 통합 + 마무리
4-1. 백엔드 ↔ 프론트엔드 실데이터 연동
4-2. 엣지케이스 처리
     (데이터 없을 때, API 오류 시 등)
4-3. 퍼포먼스 최적화
4-4. 크로스브라우저 + 모바일 테스트
4-5. GitHub Pages 최종 배포
